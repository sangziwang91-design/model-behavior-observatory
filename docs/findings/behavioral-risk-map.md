# Behavioral Risk Map

This page provides a compact public-facing map of several recurring behavioral mismatches.

It is designed as a first orientation layer.
It does not expose internal scoring logic, protected protocol details, or reconstruction paths.

## Core mismatch

The repository studies one recurring problem:

> confidence and structure can outrun securely available truth.

The findings below describe different ways that mismatch becomes visible.

## The map

### 1. Confidence Gap
A system can sound more certain than the evidence securely supports.

See: `f1-confidence-gap.md`

### 2. Structure Outruns Evidence
Explanatory shape can arrive before secure grounding.

See: `f2-structure-outruns-evidence.md`

### 3. Premature Legitimacy
Weakly grounded material can begin to feel established too early.

See: `f3-premature-legitimacy.md`

### 4. Observation, Not Remediation
The repository should be read as a behavioral observation surface, not a fix layer.

See: `f4-observation-not-remediation.md`

### 5. Silence Default
A system may continue filling the gap instead of clearly holding the boundary.

See: `f6-silence-default.md`

### 6. Boundary Retention
A model may fail to keep an unknown object as unknown.

See: `f10-boundary-retention.md`

### 7. Pseudo-Consistency
Answers can look stable while their underlying evidential handling is not.

See: `f11-pseudo-consistency.md`

### 8. Structure Drift
Explanatory structure can grow across steps without corresponding evidence growth.

See: `f12-structure-drift.md`

### 9. Confidence–Structure Gap
Confidence and structural completeness can scale faster than grounding.

See: `f13-confidence-structure-gap.md`

## How to use this page

This map is useful when you want to:

- orient a new reader quickly
- compare related behavior risks
- separate different kinds of plausibility failure
- understand why "sounds coherent" is not enough

## Correct reading boundary

This is a public-safe map.
It is not a raw lab notebook, not a reproducibility package, and not a disclosure of internal control architecture.
