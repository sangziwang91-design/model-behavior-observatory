# F3 — Premature Legitimacy

## Public finding

The most important failure mode is not always direct fabrication first.
Often it begins earlier:

> an unverified topic starts to sound like a legitimate framework, mechanism, or research direction before the evidence justifies that move.

This page refers to that shift as **premature legitimacy**.

## Why this matters

For many readers, a claim starts to feel more credible when it is:

- named clearly
- organized into a structure
- linked to familiar concepts
- explained in theory-like language

Those surface features can make an uncertain object appear more established than it really is.

## What the pattern looks like

Premature legitimacy often appears when the model:

- turns a weakly grounded object into a framework-like description
- maps unknown material into familiar academic or technical categories
- expands a speculative idea into a stable-sounding direction
- replaces boundary language with explanatory momentum

The answer may still sound careful.
That is part of why the pattern is hard to notice.

## Expansion before fabrication

A public misunderstanding is to assume the key danger begins only when the model states something obviously false.

This repository points to an earlier stage:

- the topic is not yet securely grounded
- but the answer already sounds structurally legitimate

That movement can happen before outright falsehood becomes obvious.

## Public interpretation

The problem is not only whether a sentence is true or false.
The problem is whether uncertainty is being transformed into legitimacy too early.

In other words, the model may not simply invent a fact.
It may first create the conditions under which an uncertain object feels easier to accept.

## What this repository observes

This repository does **not** claim to detect truth directly.
It observes how models behave when truth is not securely available.

From that perspective, premature legitimacy is useful as a public-facing finding because it shows how:

- structure
- fluency
- explanation
- conceptual familiarity

can combine to make weakly grounded content feel more established than it is.

## What this finding does not claim

This finding does **not** mean:

- every framework-like answer is misleading
- naming something is always a problem
- explanation itself is unsafe

It means only that under uncertainty pressure, explanatory structure can grant legitimacy faster than evidence deserves.

## Public reading rule

When an uncertain topic is explained too smoothly, the right question is not only:

- "Is this internally coherent?"

It is also:

- "Has coherence arrived before secure grounding?"
