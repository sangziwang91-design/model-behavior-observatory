# Public Evidence Chain

This repository is **not** the raw source of record.

It is the public-safe release layer.

## Release flow

Public material in this repository follows a four-layer flow:

1. **Raw source files** — original PDFs, screenshots, exports, and working materials
2. **Reviewed Notion pages** — readable internal analysis pages with risk filtering
3. **Client-safe materials** — stabilized external-facing reports with core technical details withheld
4. **GitHub public layer** — public-safe summaries, findings, and research-facing notes

## What this means in practice

- Raw PDFs do **not** publish directly to GitHub
- Internal analysis pages do **not** publish directly to GitHub
- Public pages here are **reconstructed summaries**, not raw exports
- GitHub is the public window, not the internal review console

## Why the boundary matters

The purpose of this repository is to make behavior patterns legible **without** exposing:

- internal thresholds
- trigger structures
- routing logic
- full protocol reconstruction paths
- reverse-engineerable control details

## What readers can trust here

Readers should treat the public pages in this repository as:

- evidence-linked summaries
- reviewed public-safe expressions
- observation-facing materials designed for external reading

They should **not** be treated as a full dump of internal experimentation.

## Public standard

A page is suitable for GitHub only when it is:

- non-redundant
- non-leaking
- irreversibly reconstructed
- market-readable
- safe for public circulation
