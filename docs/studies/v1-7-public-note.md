# V1.7 Public Note

## Study Scope

This note is a limited public summary of a ten-system behavioral study conducted under an internal evaluation standard. It is provided as a **research window**, not as a full technical release. The underlying methodology is proprietary, the evaluated identifiers are anonymized, and no public system mapping is provided.

## What Was Evaluated

The study examined generic aspects of model behavior in a protocol-based evaluation setting. The public-safe scope includes observable response behavior, refusal and non-execution patterns, high-level transparency limits, and cross-system differences in how models respond to structured evaluation requests.

## What Is Intentionally Withheld

The full raw report is not public. Methodology details, scoring parameters, classification algorithms, complete matrices, exact scores, system-level identifiers, and any mapping that could reconstruct private internal logic are intentionally withheld.

## High-Level Cross-Cutting Findings

First, no evaluated system demonstrated full behavioral transparency within the limited prompt-layer setting used in the study. Second, refusal and non-execution responses still carried analytical value because they exposed meaningful differences in how systems handled the evaluation request. Third, apparent execution quality did not by itself establish reliable self-knowledge or boundary awareness across systems. Fourth, several emergent behaviors suggested that protocol design can itself shape what becomes observable in a behavioral study. Fifth, these results should be read as cross-cutting observations from a limited evaluation scope rather than as stable, repeated performance conclusions.

## Limitations

This was a limited evaluation scope rather than a repeated public benchmark. Repeated stability is not established publicly, and the present note should not be interpreted as a comprehensive validation of any system, product, or deployment context.

## Why No Full Raw Report Is Public

The full report is withheld because the evaluation methodology is proprietary, the identifiers are anonymized and not mapped publicly, this is not a full release, and the results reflect a limited evaluation scope. Repeated stability is not established publicly, and a raw release would expose more structure than is appropriate for the current public boundary.
