# Illusion Map

## Overview

This page provides a bounded public summary map of several related observable behavior patterns.

The goal is not to publish hidden methods.
The goal is to show how multiple public-safe findings relate to one another.

---

## Core map

```text
system-like generation
  ↓
synthetic system scoring
  ↓
recursive self-audit illusion
  ↓
self-defense illusion
  ↓
illusion stack
  ↓
synthetic legitimacy
```

This map should be read as a progression of observable layers, not as a claim about internal architecture.

---

## Layer descriptions

### 1. System-like generation
The model produces text that looks like:

- internal process descriptions
- configuration or workflow structure
- administrator-style framing
- system or governance language

### 2. Synthetic system scoring
The model adds audit-like or verification-like scoring to system-like text.
This can make plausibility feel like validation.

### 3. Recursive self-audit illusion
The model not only generates the object, but also reviews, scores, and comments on that object.
This can create a stronger appearance of internal checking.

### 4. Self-defense illusion
The model begins to describe how such outputs should be detected, constrained, or corrected.
This adds the appearance of a defensive posture.

### 5. Illusion stack
Several synthetic legitimacy layers appear together and reinforce one another.
The result can feel much stronger than any single layer alone.

### 6. Synthetic legitimacy
The response appears legitimate through structure, language, and layered review signals even without external proof.

---

## Why this matters

These patterns are risky because readers may interpret stacked signals as evidence.

But public-safe interpretation requires a stricter distinction between:

- surface legitimacy
- external verification

A response can contain many legitimacy signals without being externally validated.

## Public use

This page supports:

- orientation for readers
- public-safe benchmark framing
- cross-model comparison planning
- discussion without exposing private methods

## Boundary

This summary map does not disclose private prompts, proprietary control structures, hidden scoring rules, or reconstruction paths.
It only organizes public-safe observable findings.
